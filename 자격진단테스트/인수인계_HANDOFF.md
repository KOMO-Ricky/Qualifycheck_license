## 1. 프로젝트 개요

개인택시 **양수(讓受) 자격 자가진단** 웹페이지 + 문자(MMS) 자동발송 시스템.

- **웹페이지**: 단일 HTML 파일 SPA. 사용자가 거주 이력·자격증·경력 등을 입력하면
  양수 자격 충족 여부를 진단하고, "진단표 이미지 + 맞춤 필요서류 목록"을 문자로 발송 신청.
- **백엔드**: Google Apps Script 웹앱이 폼 제출을 받아 (1) 구글 시트에 저장,
  (2) Solapi(솔라피) API로 MMS 발송(진단표 이미지 + 서류 목록 텍스트).
- **사용자**: 비개발자. **모든 대화·설명은 한국어로.**

## 4. 백엔드(솔라피_Code.gs) 핵심

- **doPost**: 폼 수신 → 시트 저장(rowNum 반환) → MMS 발송 접수 → 발송상태 추적용 트리거 자동 설치.
- **Script Properties(값은 각자 설정, 여기엔 이름만)**:
  `SOLAPI_API_KEY`, `SOLAPI_API_SECRET`, `SOLAPI_SENDER`(발신번호), `SHEET_ID`, (선택)`SHEET_NAME`.
- **MMS 규격**: 이미지 ≤ 200KB(별도 첨부), 텍스트 ≤ 2000바이트(≈ 한글 1000자). 초과 시 2통 자동 분할.
- **발송 결과 추적**: 접수 후 즉시 응답(fire-and-forget) → 1분 주기 시간 트리거(`updatePendingSends`)가
  그룹 상태를 폴링해 시트의 발송결과 열(성공/실패)을 갱신. 실패 사유는 `friendlyReason()`으로
  직관적 한국어(예: "없는 번호/결번")로 치환.
- **시트 컬럼(A~T)**: A접수일시(KST)~O신규교육수료여부, P상담유형, Q개인정보동의, R마케팅동의,
  S문자발송결과(요약), T문자발송결과(상세). createdAt은 `nowKST()`로 한국시간.
- 배포: Apps Script 편집기 > 배포 > 새 배포(웹앱, 액세스 "모든 사용자"). 웹앱 URL을 HTML의
  `GAS_WEB_APP_URL` 상수에 반영.

## 5. 웹페이지(개인택시진단_병합.html) 구조 메모

- 화면 전환: `go(id, pushStack)` / `goBack()` + `navStack`. 화면은 `.screen.active` 토글.
- 뒤로가기 시 해당 화면의 입력/선택을 초기화(`_SCREEN_STATES`), 드롭다운·계산기도 리셋.
- 경력 계산: 3년(36개월)/6년(72개월) 창에서 월(月) 집합 계산. **법인/기관 운전직은
  다른 경력과 합칠 때만 1/2 환산**(단일 운전직 경력은 100%). 자격 = t3≥30 OR t6≥60.
- 복수경력자가 **일부 경력만으로 충족**되면 최소 개수 조합(동률 시 개월 많은 쪽)을 찾아
  해당 경력 서류만 안내(`findEssentialTypes`, `state.docExpTypes`, `window._redundantTypes`).
- 진단표는 `html2canvas`로 이미지 생성 → 솔라피 MMS로 발송.
- 검증: Playwright/Chromium 사용(`/opt/pw-browsers/.../chrome`), 스크래치패드에 `.cjs` 스크립트 작성해
  헤더/푸터 겹침·스크롤·JS 오류를 PC(1280/1440)·모바일(360/390)에서 확인.
