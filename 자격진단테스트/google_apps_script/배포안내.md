# 구글 스프레드시트 연동 (Apps Script) 배포 안내

진단 웹페이지의 "문자 발송 신청" 폼에 입력된 **이름·연락처·진단결과·동의 여부**를
구글 스프레드시트에 자동 저장하기 위한 설정 방법입니다.

이 폴더의 `Code.gs` 코드를 스프레드시트에 연결된 Apps Script에 붙여넣고 **재배포**하면 됩니다.

---

## 이번 버전에서 바뀐 점

- 폼에 **마케팅 정보 수집·이용 동의(선택)** 항목이 추가되었습니다.
- 이에 따라 전송 데이터에 `marketingAgreed`(마케팅 동의 여부, Y/N) 항목이 추가됩니다.
- `Code.gs`는 **새 항목이 생기면 시트에 컬럼을 자동으로 추가**하도록 만들어져 있어,
  이번 한 번만 아래 절차대로 교체·재배포하면 됩니다.

---

## 처음 설정하는 경우 (또는 스크립트를 교체하는 경우)

### 1. Apps Script 편집기 열기
1. 데이터를 저장할 **구글 스프레드시트**를 엽니다.
2. 상단 메뉴 **확장 프로그램(Extensions) → Apps Script** 클릭.

### 2. 코드 붙여넣기
1. 편집기에 있던 기존 코드를 모두 지웁니다.
2. 같은 폴더의 **`Code.gs` 내용을 전부 복사**해서 붙여넣습니다.
3. 상단의 `SHEET_NAME` 값(`'신청데이터'`)을 **데이터를 기록할 탭(시트) 이름**과 똑같이 맞춥니다.
   - 해당 이름의 탭이 없으면 자동으로 첫 번째 탭에 기록됩니다.
4. 💾 **저장**(Ctrl/⌘ + S).

### 3. 웹 앱으로 배포
1. 우측 상단 **배포(Deploy) → 새 배포(New deployment)**.
2. 유형 선택(톱니바퀴 ⚙️) → **웹 앱(Web app)**.
3. 설정:
   - 설명: 아무거나 (예: `진단 신청 수집 v2026-07-02`)
   - **실행 계정(Execute as): 나(Me)**
   - **액세스 권한(Who has access): 모든 사용자(Anyone)**
4. **배포(Deploy)** → 권한 승인 요청이 뜨면 본인 계정으로 승인.
5. 배포 후 나오는 **웹 앱 URL**(`.../exec`로 끝남)을 복사합니다.

### 4. HTML에 URL 붙여넣기
- `병합결과/개인택시진단_병합.html` 파일에서 아래 부분을 찾아 URL을 교체합니다.
  ```js
  const GAS_WEB_APP_URL = 'https://script.google.com/macros/s/......./exec';
  ```
- (이미 예전 URL이 들어 있습니다. 새로 배포해서 URL이 바뀌었다면 새 URL로 교체하세요.)

---

## 이미 배포돼 있고 URL을 그대로 쓰고 싶은 경우 (권장)

기존 배포 URL을 유지한 채 코드만 최신으로 바꾸려면:

1. 위 **1~2단계**대로 `Code.gs`를 붙여넣고 저장합니다.
2. **배포(Deploy) → 배포 관리(Manage deployments)**.
3. 기존 웹 앱 배포 옆 **연필(수정) 아이콘** 클릭.
4. **버전(Version)** 을 **새 버전(New version)** 으로 선택 → **배포(Deploy)**.
   - 이렇게 하면 **URL은 그대로**이고 코드만 최신으로 갱신됩니다.
   - 즉, HTML의 `GAS_WEB_APP_URL`은 바꾸지 않아도 됩니다.

---

## 저장되는 데이터 예시 (주요 항목)

| 항목(헤더) | 설명 |
|---|---|
| createdAt | 신청 시각(ISO) |
| name | 이름 |
| phone | 연락처 |
| privacyAgreed | 개인정보 수집·이용 동의 (Y/N) |
| **marketingAgreed** | **마케팅 정보 수신 동의 (Y/N) ← 이번에 추가** |
| consultType | 양수계약가능 / 자격부족안내 |
| 운행 희망 지역 1·2단계, 거주 이력, 자격증·경력·교육 여부 등 | 진단 응답 결과 |

> 참고: 동의하지 않고 제출한 경우 발송 자체가 막혀 있어, 저장되는 행은 항상
> 개인정보 동의(privacyAgreed=Y) 상태입니다. 마케팅 동의는 선택이라 Y/N이 섞일 수 있습니다.

---

## 동작 확인 방법

1. 배포한 웹 앱 URL을 브라우저 주소창에 붙여넣어 접속 → **`OK ...`** 문구가 보이면 정상.
2. 실제 페이지에서 이름·연락처를 넣고 신청 → 스프레드시트에 새 행이 추가되는지 확인.

---

## ❗ 데이터가 스프레드시트에 안 들어올 때 (문제 해결)

아래 순서대로 하나씩 확인하세요.

### 1) 탭(시트) 이름 문제 — 가장 흔함
- 이 스크립트는 `SHEET_NAME`에 적힌 탭에 기록합니다.
- **이 버전부터는 해당 이름의 탭이 없으면 자동으로 만들어서** 거기에 기록하므로,
  코드를 최신으로 교체·재배포하면 `자격진단 응답DB` 탭이 자동 생성됩니다.
- (예전 버전) 탭 이름이 조금이라도 다르면 데이터가 **맨 앞(가장 왼쪽) 탭**에 들어갔을 수 있으니
  맨 앞 탭도 확인해 보세요.

### 2) URL 불일치
- Apps Script → **배포 → 배포 관리**에서 현재 웹 앱 URL을 복사해,
  `개인택시진단_병합.html`의 `GAS_WEB_APP_URL` 값과 **완전히 같은지** 확인하세요.
- '새 배포'로 새로 만들었다면 URL이 바뀝니다. 이 경우 HTML의 URL을 새 것으로 교체해야 합니다.
- (기존 배포를 '새 버전'으로 수정했다면 URL은 그대로 유지됩니다.)

### 3) 접근 권한
- 배포 설정에서 **액세스 권한(Who has access)** 이 반드시 **모든 사용자(Anyone)** 여야 합니다.
- '나만(Only myself)'이면 익명 방문자의 신청이 차단됩니다.

### 4) 실행 로그로 원인 정확히 보기 (가장 확실)
- Apps Script 편집기 왼쪽 메뉴 **실행(Executions)** 클릭.
- 웹페이지에서 테스트 신청을 한 뒤 목록을 보면:
  - `doPost` 항목이 **없다** → 요청이 아예 안 온 것 (→ 2번 URL / 3번 권한 확인)
  - `doPost`가 **실패(빨강)** → 클릭하면 오류 메시지가 나옵니다. 그 내용을 알려주세요.
  - `doPost`가 **완료(초록)** → 기록은 됐다는 뜻. 어느 탭에 들어갔는지 1번 확인.
