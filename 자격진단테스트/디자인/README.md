# Handoff: 개인택시 양수 자격 진단 (도시에 디자인)

## Overview
개인택시 양수 희망자가 단계별 설문에 응답해 양수 자격을 진단받고, 결과표 확인 후 전문 상담을 신청하는 웹사이트의 **전체 페이지 디자인**입니다. 본 패키지는 모든 화면(32개)을 "도시에(dossier)" 스타일로 디자인한 표준 HTML/CSS/JS 소스입니다.

## About the Design Files
`개인택시 진단 디자인소스.html` 은 **디자인 레퍼런스**입니다 — 의도한 외형/레이아웃/인터랙션을 보여주는 프로토타입이며, 그대로 프로덕션에 배포하는 코드가 아닙니다. 이 HTML 디자인을 **대상 코드베이스의 환경(React/Vue/순수 HTML 등)과 기존 패턴에 맞게 재구현**하시면 됩니다. 프레임워크가 없다면 순수 HTML/CSS/JS 그대로 채택해도 무방합니다(의존성 없음, 폰트 CDN만 사용).

페이지 간 **실제 분기 로직과 외부 DB(구글 시트) 연동은 의도적으로 포함하지 않았습니다.** 디자인 구조와 화면 전환(미리보기용)만 들어있으며, 자격 판정 알고리즘·구글시트 전송은 직접 구현하시면 됩니다.

## Fidelity
**High-fidelity (hifi)** — 최종 컬러·타이포·간격·인터랙션이 확정된 픽셀 단위 목업입니다. 아래 디자인 토큰을 그대로 사용해 재현하세요.

## 파일 구조 / 동작 방식
- 단일 파일. 좌측 `<aside class="rail">`(진행 링 + 수집정보 원장 + 페이지 미리보기 드롭다운 + 푸터) + 우측 `<main class="main">`.
- 모든 화면은 `<section class="screen" id="scr-{화면ID}">` 로 들어있고, 활성 1개만 `.active` 클래스로 표시됩니다.
- 각 화면 `<section>` 에 메타데이터 속성: `data-stage`(진행 0~8), `data-rail`(레일 단계명), `data-chip`(상단 칩), `data-back`(이전 화면 ID).
- 선택지 버튼은 `data-to="다음화면ID"`, 그리고 원장 기록용 `data-lk`(항목명)·`data-lv`(값) 속성을 가집니다. **여기에 실제 분기 로직을 연결하세요.**
- 하단 `<script>` 의 `showScreen(id)` 가 화면 전환·진행 링·원장을 갱신합니다. 실제 자격 판정/분기로 교체·확장하세요.

## Interactions & Behavior
- 화면 진입: `qIn` 페이드업 애니메이션.
- 골드 CTA 버튼: 호버 상승, 액티브 축소(scale .985), `shimmer` 광택 애니메이션(4s).
- 선택지 카드: 호버 시 골드 보더 + 상승, 액티브 축소. 복수선택(s6A4)은 `.selected`(옵시디언 채움) 토글.
- 좌측 진행 링: `data-stage/8` 비율로 stroke-dashoffset 갱신(0.7s ease).
- 원장: `data-lk/data-lv` 가진 선택지 클릭 시 항목이 누적 표시.
- 반응형: 860px 미만에서 레일이 상단 진행바(미니바+드롭다운)로 접히고 단일 컬럼 + 하단 푸터 밴드 표시.
- 상담 폼: 개인정보 동의 체크 시에만 제출 버튼 활성.

## State Management (구현 시 필요)
- `picks` (응답 누적), 현재 화면 ID, 경력 계산 결과(3년/6년 무사고 개월수), 자격 판정 결과, 폼 입력값/동의 여부.
- 전환 트리거: 선택지 클릭, 다음/이전 버튼, 페이지 picker(개발 중 점검용 — 배포 시 제거 가능).
